## Commands

### Development and Build
- Run local development server (with auto-seeding): `npm run dev` (note: script uses `pnpm seedLocalD1` internally, so ensure `pnpm` is installed globally or adjust as needed)
- Typecheck and dry-run deploy: `npm run check`
- Generate Environment Types (Wrangler Typegen): `npm run cf-typegen`

### Database & Migrations (Cloudflare D1)
- Apply migrations locally: `npm run seedLocalD1` (translates to `wrangler d1 migrations apply DB --local`)
- Apply migrations to remote production DB: `npm run predeploy` (translates to `wrangler d1 migrations apply DB --remote`)
- Create a new migration file: `npx wrangler d1 migrations create d1-db <migration_name>`

### Deployment
- Deploy to Cloudflare Workers: `npm run deploy` (automatically runs `predeploy` to apply remote migrations first)

*Note: No dedicated linting or testing frameworks (such as Jest or Vitest) are configured in this codebase.*

## High-Level Architecture

This project is a Cloudflare Workers application integrated with a Cloudflare D1 Serverless SQL database.

### Core Architecture Components

1. **Worker Entrypoint (`src/index.ts`)**:
   - Implements the standard Cloudflare Workers `ExportedHandler<Env>` fetch handler.
   - Accesses the D1 database binding via `env.DB`.
   - Executes queries (e.g., `SELECT * FROM comments LIMIT 3`) and forwards results to the HTML renderer.

2. **UI Renderer (`src/renderHtml.ts`)**:
   - A simple presentation utility that returns static HTML injected with query result data.

3. **Database Schema & Migrations (`migrations/`)**:
   - Database schemas are managed using SQL migration files under the `migrations/` directory (e.g., `0001_create_comments_table.sql`).
   - These are sequentially numbered and applied either locally or remotely using wrangler.

4. **Configuration & Bindings (`wrangler.json`)**:
   - Defines worker settings, compatibility date, entrypoint, and database bindings.
   - The D1 database binding is configured under `d1_databases` with binding name `DB`.

5. **Type Safety (`worker-configuration.d.ts` & `tsconfig.json`)**:
   - Types are generated by Wrangler to map the bindings in `wrangler.json` to the global `Env` interface.
